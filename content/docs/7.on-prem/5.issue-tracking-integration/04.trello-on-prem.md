Unlike the Cloud version, ZeroThreat On-Prem connects to Trello using an **API Key** and **Token**, and operates entirely from within your controlled infrastructure.

::hint{ style="background-color:#fff4e6; color:#f97316; border:1px solid #f6d2b0;" icon="triangle-exclamation" iconClass="text-[#f97316]"}
###### ZeroThreat On-Prem requires outbound network access to Trello. Ensure your firewall allows API communication to Trello endpoints.
::


## Prerequisites

Before you begin:

* Your target is associated with an **Enterprise plan** that supports Issue Tracking.
* You have permission in Trello to create cards in the intended board and list.
* Your On-Prem deployment has network access to Trello.


### Step 1: Generate Trello API Credentials

ZeroThreat On-Prem requires a Trello **API Key** and **Token** for authentication.

Follow these steps:

1. Log in to your Trello account.
2. Go to `https://trello.com/power-ups/admin`.
3. Under the **Apps** section, click **New**.

::fiqure-img{source="/Images/image (428).png"}
::

4. Fill in the required details for your app and click **Create**.
5. Once the app is created, open the **API Key** section.
6. Click **Generate a new API key**.

::fiqure-img{source="/Images/image (429).png" width="90%"}
::

7. Next to the generated API key, click **Token** to generate a token.

::fiqure-img{source="/Images/image (430).png" width="90%"}
::

8. Authorize the app by clicking **Allow**.

::fiqure-img{source="/Images/image (431).png"}
::

9. Copy and securely save the generated **API Key** and **Token**.

::fiqure-img{source="/Images/image (432).png"}
::

::hint{style="background-color:#f7fcff; border:1px solid #07405a33; color:#1e6995;" icon="circle-info" iconClass="text-sky-600"}
###### The steps above may vary slightly depending on your Trello account or workspace setup. In general, you simply need to generate a **Trello API Key** and **Token** from your Trello developer settings.
::


### Step 2: Configure Trello in ZeroThreat On-Prem

1. Navigate to the **Targets** (<span><img src="/Images/image (44).png" alt="" class="w-6 inline-block"></span>) section.
2. Select the target you want to configure and open the **Target Configuration** page.
3. Go to the **Issue Tracking** (<span><img src="/Images/image (208).png" alt="" class="w-24 inline-block"></span>) section.
4. Locate **Trello** and click **Configure**.
5. In the popup, enter your **Trello API Key** and **Trello API Token** copied in Step 1.

::fiqure-img{source="/Images/image (433).png"}
::

6. Click **Save**.

Your Trello integration is now connected to the target.


### Step 3: Create Trello Cards from a Scan Report

Once Trello is configured:

1. Go to the **Scans** (<span><img src="/Images/image (210).png" alt="" class="w-6 inline-block"></span>) section.
2. Open a completed scan report.
3. Click the **Trello Card** button at the top of the report.

::fiqure-img{source="/Images/image (434).png"}
::

A Trello configuration popup will appear.


### Step 4: Configure Trello Card Settings

In the Trello popup:

1. Select the **Workspace**.
2. Choose the **Board** where vulnerabilities should be created.
3. Select the **List** where cards should be added.

::fiqure-img{source="/Images/image (435).png"}
::

4. Click **Submit** to proceed.

All cards created from that scan will be added to the selected Trello board and list.


### Step 5: Create Cards for Vulnerabilities

1. In the scan report, identify the vulnerability you want to track.

::fiqure-img{source="/Images/image (436).png" width="90%"}
::

2. Click **Create Issue**.

ZeroThreat will generate individual Trello cards for each finding under that vulnerability.

All cards are immediately pushed to Trello and can be managed directly within your Trello board.

::hint{style="background-color:#f7fcff; border:1px solid #07405a33; color:#1e6995;" icon="circle-info" iconClass="text-sky-600"}
###### Create Trello cards only for validated and high-priority vulnerabilities to avoid cluttering your boards.
::


## Viewing and Managing Issues

After creating cards:

* Go to **Created Issues** inside the scan report to see all Trello cards generated.
* Click any card to open it directly in Trello.

::fiqure-img{source="/Images/image (437).png"}
::

Each Trello card includes:

* Detailed vulnerability description
* HTTP request and response evidence
* Proof-of-concept details
* Direct link back to the full ZeroThreat scan report

::hint{ style="background-color:#fff4e6; color:#f97316; border:1px solid #f6d2b0;" icon="triangle-exclamation" iconClass="text-[#f97316]"}
###### In ZeroThreat On-Prem, bi-directional comment synchronization and automatic issue status synchronization are not available. Changes made in Trello will not automatically reflect back in ZeroThreat portal.
::


## Best Practices

* Use a dedicated Trello API Key and Token for ZeroThreat integration.
* Rotate tokens periodically as part of your security practices.
* Create cards only for validated vulnerabilities to keep boards manageable.
* Ensure your firewall consistently allows communication to Trello endpoints.
