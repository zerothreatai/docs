description: ZeroThreat On-Prem allows you to create Jira issues directly from scan findings, enabling security and development teams to collaborate within existing sprint workflows. By integrating Jira with your On-Prem deployment, validated vulnerabilities can be converted into structured tasks inside your Jira projects with complete technical context.
---

Unlike the Cloud version, ZeroThreat On-Prem connects to Jira using an API token and operates entirely from within your controlled infrastructure.

::hint{ style="background-color:#fff4e6; color:#f97316; border:1px solid #f6d2b0;" icon="triangle-exclamation" iconClass="text-[#f97316]"}
###### ZeroThreat On-Prem requires outbound network access to your Jira instance (e.g., `https://your-domain.atlassian.net`). Ensure your firewall allows API communication.
::


## Prerequisites

Before you begin:

* Your target is associated with an Enterprise plan that supports Issue Tracking.
* You have permission in Jira to create issues in the intended project.
* Your On-Prem deployment has network access to your Jira domain.


### Step 1: Generate a Jira API Token

ZeroThreat On-Prem requires a Jira API token for authentication.

Follow these steps:

1. Log in to your Jira account.
2. Click on your profile avatar (top right) and select **Manage account**.
3. Navigate to **Security** and under **API tokens**, click **Create and manage API tokens**.

::fiqure-img{source="/Images/image (402).png" width="90%"}
::

4. Click **Create API Token** (<span><img src="/Images/image (403).png" alt="" class="w-24 inline-block"></span>)
5. Provide a name for the token and set an expiry date.

::fiqure-img{source="/Images/image (404).png"}
::

6. Click **Create** and copy the generated token (it will only be shown once).

::fiqure-img{source="/Images/image (405).png"}
::

7. In addition to the API token, you will need:

* Your **Jira domain URL**\
  Example: `https://your-company.atlassian.net/`
* The **email address** associated with your Jira account.

::hint{style="background-color:#f7fcff; border:1px solid #07405a33; color:#1e6995;" icon="circle-info" iconClass="text-sky-600"}
###### The steps above may vary slightly depending on your Jira account. In general, you simply need to obtain a **Jira API Token,** your **Jira Domain URL**, and the **email address** associated with your Jira account, using any method available in your Jira settings.
::


### Step 2: Configure Jira in ZeroThreat On-Prem

1. Navigate to the **Targets** (<span><img src="/Images/image (44).png" alt="" class="w-6 inline-block"></span>) section.
2. Select the target you want to configure and open the **Target Configuration** page.
3. Go to the **Issue Tracking** (<span><img src="/Images/image (208).png" alt="" class="w-24 inline-block"></span>) section.
4. Locate **Jira** and click **Configure**.
5. In the popup, enter the **Jira Domain URL**, your **Jira Account Email**, and the **Jira API Token**.

::fiqure-img{source="/Images/image (406).png"}
::

6. Click **Save**.

Your Jira integration is now connected to the target.


### Step 3: Create Jira Issues from a Scan Report

Once Jira is configured:

1. Go to the **Scans** (<span><img src="/Images/image (210).png" alt="" class="w-6 inline-block"></span>) section.
2. Open a completed scan report.
3. Click the **Jira Task** button at the top of the report.

::fiqure-img{source="/Images/image (407).png"}
::

A Jira configuration popup will appear.


### Step 4: Configure Jira Issue Settings

1. In the Jira Task popup:

* Select your Jira instance.
* Choose the **Project** where vulnerabilities should be created.
* Select the **Board**.
* Choose the **Sprint** (optional).

::fiqure-img{source="/Images/image (408).png"}
::

2. Click **Submit** to proceed.

All issues created from that scan report will be added to the selected project and sprint.


### Step 5: Create Issues for Vulnerabilities

1. In the scan report, identify the vulnerability you want to track.

::fiqure-img{source="/Images/image (409).png" width="90%"}
::

2. Click **Create Issue**.

ZeroThreat will generate individual Jira tickets for each finding under that vulnerability.

All tickets are immediately pushed to Jira and can be managed directly inside your Jira board.

::hint{style="background-color:#f7fcff; border:1px solid #07405a33; color:#1e6995;" icon="circle-info" iconClass="text-sky-600"}
###### Create Jira issues only for validated and high-priority vulnerabilities to avoid cluttering your backlog.
::


## Viewing and Managing Issues

After creating issues:

* Go to **Created Issues** inside the scan report to see all Jira tickets generated.
* Click any issue to open it directly in Jira.

Each Jira ticket includes:

* Detailed vulnerability description
* HTTP request and response evidence
* Proof-of-concept details
* Direct link back to the full ZeroThreat scan report

::fiqure-img{source="/Images/image (410).png"}
::

::hint{ style="background-color:#fff4e6; color:#f97316; border:1px solid #f6d2b0;" icon="triangle-exclamation" iconClass="text-[#f97316]"}
###### In ZeroThreat On-Prem, bi-directional comment synchronization and automatic issue status synchronization are not available. Changes made in Jira, such as updating status or adding comments, will not automatically reflect back in ZeroThreat.
::


## Best Practices

* Use a dedicated Jira API token for ZeroThreat integration.
* Rotate API tokens periodically as part of security hygiene.
* Align issue creation with sprint planning cycles.
* Ensure your firewall consistently allows communication to your Jira domain.
