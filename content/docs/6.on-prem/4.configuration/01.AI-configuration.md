## Introduction

ZeroThreat uses AI-driven capabilities across multiple areas of the platform to improve clarity, reduce manual effort, and accelerate security analysis. These capabilities go beyond a single feature and are used throughout the platform wherever contextual understanding or summarization is required.

AI-powered features in ZeroThreat include, but are not limited to:

* Executive summaries for scan results
* Remediation guidance and fix recommendations
* Vulnerability explanations and contextual insights
* Additional analysis-driven enhancements across the platform

In an **On-Prem** deployment, ZeroThreat gives you control over how AI is enabled by allowing you to connect your own AI providers. This ensures that AI usage aligns with your internal security policies, compliance requirements, and data handling expectations.

***

## Supported AI Providers

ZeroThreat On-Prem currently supports the following AI providers, with additional options planned.

Google Gemini can be integrated using an API key generated from Google AI Studio.

### Azure OpenAI

Azure OpenAI integration allows you to use OpenAI models hosted within [**Microsoft Azure OpenAI Service**](https://azure.microsoft.com/en-us/products/ai-foundry/models/openai), enabling organizations to leverage AI-driven capabilities while staying aligned with Azure governance, identity, and compliance controls. This is ideal for enterprises that require strict data residency, access control, and auditing within their Azure environment.

**Configuration steps**

1. Log in to the **Microsoft Azure Portal** and open your **Azure OpenAI** resource
2. From the resource settings, retrieve:

   * An **API Key** (for example, from the *Keys and Endpoint* section)

<figure><img src="https://1825008717-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2Fs6Y7hKb1RwZWFZo4EnUm%2Fuploads%2FpGJO0E76vFky1wgIQY09%2Fimage.png?alt=media&#x26;token=e35590e4-2264-4161-a16d-c350747d6352" alt="" width="563"><figcaption></figcaption></figure>

   * The **Endpoint URL** associated with the Azure OpenAI resource

<figure><img src="https://1825008717-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2Fs6Y7hKb1RwZWFZo4EnUm%2Fuploads%2F8aWZBFN2fX8pTikpiysb%2Fimage.png?alt=media&#x26;token=0a61156b-e645-4016-b878-a4e0546db981" alt="" width="563"><figcaption></figcaption></figure>

::hint{style="background-color:#f7fcff; border:1px solid #07405a33; color:#1e6995;" icon="circle-info" iconClass="text-sky-600"}

###### The steps above describe a generic approach to retrieving the required credentials.You only need an **API Key** and an **Endpoint URL** from your Azure OpenAI account. How you obtain these may vary depending on your Azure setup, and workflows.
::

3. Log in to your **ZeroThreat On-Prem** portal
4. Navigate to **Organization Settings** and click on **AI Configuration**

<figure><img src="https://1825008717-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2Fs6Y7hKb1RwZWFZo4EnUm%2Fuploads%2FKxYYHZYPQgaDQn2VkoOH%2Fimage.png?alt=media&#x26;token=fd74b499-645a-4748-ac53-73d5851afe6a" alt="" width="246"><figcaption></figcaption></figure>

5. Under **Azure OpenAI** Provider, toggle Enable AI Configuration (<img src="https://1825008717-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2Fs6Y7hKb1RwZWFZo4EnUm%2Fuploads%2F4s2LsjEQztAbbPj33Dyr%2Fimage.png?alt=media&#x26;token=35dff6cd-8ebc-4225-a18f-ddbe8812bfa0" alt="" data-size="line">) option
6. Enter the retrieved API key and the Endpoint URL in the fields.

<figure><img src="https://1825008717-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2Fs6Y7hKb1RwZWFZo4EnUm%2Fuploads%2FNLjUxxFSqkFRA3uekXIU%2Fimage.png?alt=media&#x26;token=9dfe9aaf-cc7d-4f13-9d40-712f661f32b5" alt="" width="563"><figcaption></figcaption></figure>

7. Save the configuration

Once saved, ZeroThreat will use the Azure-hosted OpenAI models associated with your resource to power AI-driven features across the platform.

::hint{ style="background-color:#fff4e6; color:#f97316; border:1px solid #f6d2b0;" icon="triangle-exclamation" iconClass="text-[#f97316]"}

###### AI-driven features will not function if the API key is invalid or revoked, the endpoint URL is incorrect or the Azure quotas or usage limits are exhausted.
::

**Official References**

* Microsoft Foundry documentation:[**https://learn.microsoft.com/en-gb/azure/ai-foundry/?view=foundry-classic**](https://learn.microsoft.com/en-gb/azure/ai-foundry/?view=foundry-classic)
* Connect to Azure AI Search using keys: [**https://learn.microsoft.com/en-us/azure/search/search-security-api-keys?tabs=portal-find%2Cportal-use%2Cportal-query**](https://learn.microsoft.com/en-us/azure/search/search-security-api-keys?tabs=portal-find%2Cportal-use%2Cportal-query)
