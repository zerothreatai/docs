---

This is useful when you want to confirm exposure to a known vulnerability, validate a specific attack template, or quickly test whether a target is affected by a particular security check.

::hint{ style="background-color:#f7fcff; border:1px solid #07405a33; color:#1e6995;" icon="circle-info" iconClass="text-sky-600"}
###### Attack Validation is available on Professional Cloud, Enterprise Cloud, and Enterprise On-Prem plans.
######
###### The selected target must also be linked to a subscription plan that supports Attack Validation. If the target is not linked to an eligible plan, the quick test cannot be started.
######
###### Please refer to the [**ZeroThreat Plans**](/docs/plans/zerothreat-plans 'mention') for more details.
::

## Open Attack Validation

From the **Dashboard**, click the **Attack Validation** icon in the top-right side of the page.

::fiqure-img{source="/Images/attack-validation-1.png"}
::

This opens the **Attack Validation** section, where you can view the available CVE-based templates and Open Attack Templates.

::fiqure-img{source="/Images/attack-validation-2.png"}
::

## View Available Tests

The Attack Validation section shows a list of available tests that can be run against your targets. These include CVE-based templates and Open Attack Templates.

Browse the list and choose the test you want to validate against a target.

## Run a Test Against a Target

To run a quick validation test:

1. Hover over the CVE or Open Attack Template you want to test.
2. Click **Run this Test**.

::fiqure-img{source="/Images/attack-validation-3.png"}
::

3. Select the target you want to test from the popup.

::fiqure-img{source="/Images/attack-validation-4.png"}
::

4. Once ZeroThreat confirms that the selected target is linked to an eligible subscription plan, click **Start Quick Scan**.

ZeroThreat will then run a focused quick test against the selected target to check whether it is vulnerable to the selected template.

## View Quick Test Results

After the quick scan starts, a result drawer will open. This drawer shows whether the selected target is **Vulnerable** or **Not Vulnerable** for the selected test.

::fiqure-img{source="/Images/attack-validation-5.png"}
::

If the target is vulnerable, click the result to view the validation details. The detailed view includes the HTTP request, HTTP response, and highlighted evidence showing how the vulnerability was confirmed.

::fiqure-img{source="/Images/attack-validation-6.png"}
::

Use this evidence to understand the validation result and support remediation or further investigation.

## Next Steps

Once a quick validation test is complete, review the result and evidence to decide the next action. If the target is vulnerable, share the HTTP request, response, and highlighted evidence with your development or security team for remediation.

If you want broader security coverage beyond a single validation test, run a complete scan using a Scan Profile.
